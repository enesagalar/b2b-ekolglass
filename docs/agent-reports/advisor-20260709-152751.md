# EkolGlass B2B Advisor Report

## 1. En mantikli sonraki 5 gelistirme

1. `/katalog` fiyat/stok gorunurlugunu role gore kapat/ac: `src/app/katalog/page.tsx`, `src/lib/auth.ts`, `src/domain/roles.ts`. Guest/PENDING sadece urun ve sade stok gormeli; `price.read` izni olan bayi/admin fiyat gormeli.
2. Uyumluluk/OEM CRUD ekle: `prisma/schema.prisma` icinde `ProductCompatibility` hazir; eksik olan `src/domain/validation.ts`, `src/features/catalog-management/actions.ts`, `src/app/admin/urunler/[id]/page.tsx`.
3. Kategori ve fiyat listesi yan panellerini alt ekranlara bol: mevcut yogunluk `src/app/admin/urunler/page.tsx` icinde. Oneri: `src/app/admin/urunler/kategoriler/page.tsx`, `src/app/admin/urunler/fiyat-listeleri/page.tsx`.
4. Medya karar modelini netlestir: `MediaAsset.isActive` var, hard delete riskli. Ilk adim soft pasiflestirme/aktif filtre olsun: `src/features/catalog-management/actions.ts`, `src/app/admin/urunler/[id]/page.tsx`.
5. Bayi basvuru inceleme/onay ekranini baslat: dashboard ve kayit alma var, admin ekrani yok. Dokunulacak yerler: `src/features/dealer-applications/actions.ts`, yeni `src/app/admin/bayi-basvurulari/page.tsx`, `src/features/admin/admin-shell.tsx`.

## 2. Bulunan riskler / test aciklari

- `src/app/katalog/page.tsx` su an ilk fiyat satirini herkese gosterebiliyor; bu B2B fiyat gizliligi icin ana risk.
- `ProductCompatibility` modeli var ama action/schema yok; uyumluluk sekmesi sadece okuma yapiyor.
- `scripts/smoke-admin.mjs` sadece login/dashboard kontrol ediyor; urun detay, medya, katalog filtre, fiyat/stok gorunurluk smoke kapsami disinda.
- Testler agirlikla domain helper seviyesinde: `src/domain/catalog.test.ts`, `src/domain/roles.test.ts`. Server action ve DB transaction davranislari icin test yok.
- `src/app/admin/urunler/page.tsx` cok fazla sorumluluk tasiyor; urun olusturma, listeleme, kategori, fiyat listesi ve inline duzenleme ayni dosyada.

## 3. Ana ajanla entegrasyon onerisi

Once fiyat/stok gorunurlugu lane'i ana ise alinmali; cunku public katalogdaki veri sizintisi riski diger UX islerinden daha kritik. Ardindan ayni turda kucuk kapsamla OEM action + form eklenebilir. Buyuk refactor olan kategori/fiyat alt ekranlari ayri commit/tur olmali; mevcut urun akisini kirmadan tasinmali.

Kabul kontrolu: `npm run check`, ardindan `/katalog`, `/admin/urunler`, `/admin/urunler/[id]?tab=uyumluluk`, `/admin/urunler/[id]?tab=medya` icin HTTP/browser smoke.
